Features

Built for engineers,
not drag-and-drop.

Every design decision in Mantle optimizes for developer experience, operational reliability, and zero vendor lock-in.

>_
IaC Lifecycle
Validate offline, plan diffs against running state, apply versioned definitions. Treat workflows like Terraform treats infrastructure.
validate → plan → apply

Bring Your Own Keys
Use your own API keys for OpenAI, Anthropic, or any OpenAI-compatible provider. No proxy. No markup. Full control.

Checkpoint & Resume
Every step checkpoints to Postgres. On crash, execution resumes from the last completed step. No lost work, no repeated calls.
crash recovery

Secrets as Opaque Handles

How It Works

Four commands.
That's the whole workflow.

Mantle follows the same lifecycle as infrastructure-as-code tools. If you know Terraform, you already know Mantle.

1

Validate

Offline schema validation, CEL expression checking, and connector reference resolution. Catch errors before they hit production.

mantle validate workflow.yaml
2

Plan

Diff your local definition against the applied version. See exactly what will change before committing. No surprises.

mantle plan workflow.yaml
3

Apply

Apply the versioned workflow definition. Each apply creates an immutable version, giving you full rollback capability.

mantle apply workflow.yaml
4

Run

Execute the workflow with inputs. Steps checkpoint automatically. Monitor progress with real-time logs.

mantle run my-workflow --input url=...
workflow.yaml
name: fetch-and-summarize description: Fetch data and summarize with an LLM inputs: url: type: string description: URL to fetch steps: - name: fetch-data action: http/request params: method: GET url: "{{ inputs.url }}" - name: summarize action: ai/completion params: provider: openai model: gpt-4o prompt: | Summarize this data: {{ steps.fetch-data.output.body }}
